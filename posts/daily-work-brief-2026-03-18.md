# 今日工作计划（2026-03-18）

## 一、今日总目标与验收导向
今天所有工作统一围绕一个核心目标推进：**为周日的“一键部署可验收”做倒排闭环**。我们不是追求“并行开更多线”，而是要把多条任务线收敛到同一验收结果上，确保在一台独立机器上，能够完成“小虾 + 地图在线版”的可部署、可运行、可演示、可回放。

本次周日验收的口径明确如下：
1. 在独立机器完成一键部署；
2. 可跑通统一后的核心 Story；
3. 部署后仅需配置三项参数：通信端口、模型服务 URL、模型 Key；
4. 除上述配置外，其他能力默认可运行。

这意味着今天每个任务都必须回答一个问题：**它对周日验收的直接贡献是什么**。无法回答或贡献间接、可替代的任务，统一降级处理。

## 四、DataHub 方向：先有路线，再做扩展

今天你要启动的数据Hub研究，是接下来提升 story 上限的关键。当前判断非常明确：仅靠内部数据，无法支撑产品场景深度，必须走“内部 + 外部”数据融合路径。

### DataHub MVP 模块建议
1. 接入层：内部数据与外部数据连接器；
2. 标准化层：统一实体、事件、时空、来源字段；
3. 治理层：敏感信息匿名化、权限审计；
4. 服务层：检索、聚合、场景化接口；
5. 编排层：向编排 Agent 暴露能力目录。

- 先跑通可演示闭环，再扩展数据深度。

今日产出建议：
- 《DataHub v0.1 模块草图》；
- 《Story-数据要素映射表 v0.1》；
- 首批接入清单（内部3 + 外部2）；

  - 已接入数据源、待接入数据源、阻塞原因；

  - 字段映射与数据质量（时效/完整性/准确性）快照。

## 五、安全特性专项（新增，今日重点）

今天新增一个高优先级专题：系统化规划安全相关特性，并形成可执行的落地清单。考虑到团队对安全感知不一致，该项由你亲自主导框架与红线，同时安排主线负责人负责落地执行与验收闭环。

### 安全五条红线（今日需拍板）
1. **数据边界**：明确哪些数据可入模、可出图、可导出；
2. **权限边界**：谁可看、谁可调、谁可改配置，最小权限原则；
3. **敏感信息治理**：匿名化/脱敏策略、误伤回滚、例外审批；
4. **外部依赖控制**：关键能力优先本地化，降低公网依赖风险；
5. **审计与追溯**：关键查询、配置变更、数据调用必须可留痕。

### 分工机制
- **你（Owner）**：定义安全框架、红线与验收门槛；
- **主线负责人（Executor）**：输出《安全特性落地清单 v1》，完成故事线安全验收映射。

### 今日产出要求
- 《安全特性优先级矩阵（P0/P1/P2）》；
- 《周日验收安全门槛》；
- 《安全特性落地清单 v1》（含功能项、场景验收、部署安全）。
